Starting DISCOVER

Setting up your environment

Each client is dependent upon a valid DISPLAY.  Be sure to set the display to a valid X server before attempting to invoke them.


Users have a choice of three methods to invoke DISCOVER model clients:

Put the path to the DISCOVER launch scripts in the shell’s PATH environment variable.
csh: setenv  PATH  ${PATH}:/net/psales/opt/DISCOVER/DISIM/bin
sh/ksh: set PATH=${PATH}:/net/psales/opt/DISCOVER/DISIM/bin; export PATH

Create unique aliases for each of the clients (described below)

Type full path names as shown in next sections.

Starting DeveloperXpress

Set the ClearCase view to the private work view

Start the DeveloperXpress client by typing:

/net/psales/opt/DISCOVER/DISIM/bin/StartDevx.sh


If a user wishes to use DeveloperXpress to check-in and check-out sources, there is only one step in the Service Chooser that need to be taken.   The Service Chooser may come up automatically or it can be brought up by selecting “File”->”Service” from DeveloperXpress.  Once in the Service Chooser window, the user must select “clearcase” in the drop down list labeled “CM”.


Always hold down shift and left-click the mouse in emacs to start the DeveloperXpress window after using the service chooser (even when it says “Launching Developer Xpress … “).


By default, DeveloperXpress uses the first occurrence of emacs in the user’s path.  This can be overridden by specifying the full path to emacs in the environment variable DIS_EMACS.  This variable also allows the user to specify any command line options to emacs.


Developers that wish to use vi as opposed to emacs, should set the environment variable DIS_VIM to “1”.  The keywords in vi that the user types after a colon, “:”, are as follows:

desc display the description of the entity at the bottom of the window.

opendef jump to the definition of the entity.

query put the entity into the DeveloperXpress window.

inst bring up instances window displaying all occurences of the entity.

When an entity is highlighted, it is best to hit the Escape key before typing any vi commands.


The client is written in TCL/TK and usually takes up less than 10Mbytes on the client machine.

Starting the DISCOVER Browser

Set the ClearCase view to the private work view

Start the DISCOVER Browser by typing:

/net/psales/opt/DISCOVER/DISIM/bin/StartBrowser.sh <VERSION>


The <VERSION> argument specifies which version of the DISCOVER model to look at.  Versions are named after the application with a date stamp telling when the model build was performed.  A list of all versions will be displayed if the argument is omitted.


The Browser takes 50Mbytes and more depending on what the Engineer is using.


Adding DISCOVER Fonts to the X-Server

This is NOT a necessary step to run either version of DISCOVER but it does prove to be handy and sometimes a time saver to be able to, at a glance, see what type an entity is.


On the local machine that will be displaying the window (not necessarily the machine that runs discover) type the following command:

xset fp+ /net/psales/opt/DISCOVER/discover/lib/fonts/sun5


The next time DISCOVER is invoked, additional icons should show up in the entity lists.
