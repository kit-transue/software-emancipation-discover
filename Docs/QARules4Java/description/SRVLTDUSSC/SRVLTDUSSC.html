Servlet API - Do not use SingleThreadModel in a Servlet class

javax.servlet.SingleThreadModel is a tag interface that a servlet can implement to transfer its re-entrance problem to the servlet engine. As such, javax.servlet.SingleThreadModel is part of the J2EE specification. The application server's servlet engine handles the servlet's re-entrance problem by creating separate servlet instances for each user. Because this causes a great amount of system overhead, javax.servlet.SingleThreadModel should be avoided.

Wrong

package com.swemancipation.servlet;

import javax.servlet.http.*;
import java.beans.*;

public class STM extends HttpServlet implements SingleThreadModel {
  // some code here.
}

Tip: javax.servlet.SingleThreadModel has been used typically to protect updateable servlet instances in a multithreaded environment. The better idea is to avoid using servlet instance variables that are updated from the servlet's service method.
