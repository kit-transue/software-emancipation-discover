UITI - Use instanceof Tests to an Interfaces

Interface-based design is in general a good thing, because it allows for the flexible inclusion of different implementations.

Wrong

public class UITI {
  private void method (Object o) {
    if (o instanceof ClassBase) { }
  }
}

class ClassBase {}

Tip: Whenever possible, perform "instanceof" tests to an interface rather than a clas s.

Right

public class UITI {
  private void method (Object o) {
    if (o instanceof InterfaceBase) { }
  }
}

interface InterfaceBase {}

REFERENCE

Graig Larman, Rhett Guthrie: "Java 2 Performance and Idiom Guide" Prentice Hall PTR, 2000. pp.207
