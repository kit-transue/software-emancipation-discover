UIOE - Unnecessary 'instanceof' Evaluations

Checks that the runtime type of the left-hand side expression is the same as the one specified on the right-hand side.

Wrong

class UIOE {
    void operation () {
        Animal animal;
        Monkey monkey;
        if ( animal instanceof Animal ) {
            doSomething1(animal);
        }
        if ( monkey instanceof Animal ) {
            doSomething2(monkey);
        }
    }
}
class Animal {}
class Monkey extends Animal {}

Tip: Remove unnecessary checks

Right

class UIOE {
    void operation () {
        Animal animal;
        Monkey monkey;
        doSomething1(animal);
        doSomething2(monkey);
     }
}
class Animal {}
class Monkey extends Animal {}
