PTCOL - Place "Try/Catch" blocks Outside of Loops.

Placing try/catch blocks inside loops can slow down the execution of code. If the JIT compiler is turned off or if you are running a JVM without a JIT, performance speed may decrease by about 21 percent.

Wrong

public class PTCOL {
  void oper (FileInputStream fis) {
    for (int i = 0; i < size; i++) {
      try {
        _sum += fis.read();
      } catch (Exception e) {}
    }
  }
  private int _sum;
}

Tip: Place try/catch block outside of the loop.

Right

public class PTCOL {
  void oper (FileInputStream fis) {
    try {
      for (int i = 0; i < size; i++) {
        _sum += fis.read();
      }
    } catch (Exception e) {}
  }
  private int _sum;
}
