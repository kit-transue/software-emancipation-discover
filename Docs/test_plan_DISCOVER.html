FUNCTIONAL AREAS FOR SYSTEM TESTING


The following is a grid of the various elements of Discover that require testing.   The introduction of additional languages makes the amount of time required for complete testing combinatoric.  Needless to say exhaustive testing becomes extremely expensive:

Total time for exhaustive testing all functionality 

              on all platforms for all languages and manually:                         ~409.5 hours

               Total time for exhaustive testing the most important functionality

                            on Unix platforms using DISCOVER regression tests:                         ~129.75 hours     

 Total time for exhaustive testing all functionality on all platforms 

              for all languages using manually and regression tests:                          ~279.75 hours

In order to achieve solid coverage and acceptable time the following plan was created.  On the most significant platforms get exhaustively tested for one language.   All remaining platforms receive general testing for a single language.  “General” indicates functionality that is commonly used and central to the product.  All remaining languages are tested against only that functionality which particularly sensitive to changes in the underlying IM data structures.  The time required is as follows:

              Total time for a single pass testing on two platforms for one language
