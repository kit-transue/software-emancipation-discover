Entity and Relationship Queries

The most familiar queries are those that are used in scan mode queries, or those not about the project structure but about the project contents and the relationships between entities that make up the projects:

defines and uses
defines /ttt_shared
uses main

defines and uses take a selector: a list of data kind(s) prefixed with “-”. See the table of kind attributes in AST_node Attributes. At present, only the first three characters are significant, meaning that it is possible to use queries that are similar to the Browser. The selector -all, which returns all kinds, is the default.

uses -functions main

However, it is best to use the names listed in the kinds table.

Multiple kinds may be specified:

uses -macro -funct main

1 (funct) fprintf

2 (macro) NULL

3 (funct) fopen

4 (funct) write_ct_file

5 (funct) getenv

6 (funct) strcpy

7 (macro) stderr

8 (funct) fclose

9 (funct) exit

10 (funct) read_mt_file

11 (funct) strcat

Note: Non-symbol kinds (data cells, AST nodes, instances, and so on.) are not indexed, and so uses/defines queries on these kinds does not return results.
