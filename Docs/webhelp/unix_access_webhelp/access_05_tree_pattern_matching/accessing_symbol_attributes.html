Accessing Symbol Attributes

Recall that ast_nodes are different from symbols, and many symbol attributes return empty when applied to ast_nodes. This is true for the args attribute, for example:

apply args main
(int,char**)
apply args [data node main]
{}
apply args [data symbol [data node main]]
(int,char**)

When creating an attribute expression that applies to ast_nodes, it is often useful to be able to access the attributes for the symbol that corresponds to the data node. You can query symbol attributes by preceding the attribute name with “s.”: this converts the ast_node to a symbol (temporarily) and returns the value of the attribute:

apply s.args [data node main]
(int,char**)
