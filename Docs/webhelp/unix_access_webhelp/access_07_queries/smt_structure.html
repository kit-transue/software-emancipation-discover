SMT Structure

The SMT is created as a result of a parse-phase, model build process. In contrast to the rich, relationship-bearing structure of the AST, the SMT is flat, maintaining only an inventory of where each entity exists in your source and where it is defined. Because the SMT records the line number and offset of each symbol of your source, you can see your code as it exists prior to preprocessing and learn more about how preprocessing has affected your source.

For example, the SMT displays the existence of multiple #ifdef conditions in your source even when the #ifdef resolution that occurs during preprocessing resolves to one; while the AST contains the one symbol, the SMT helps you to understand the rest of the #ifdef construct. This is especially helpful while using DIScover for tasks such as code comprehension.
