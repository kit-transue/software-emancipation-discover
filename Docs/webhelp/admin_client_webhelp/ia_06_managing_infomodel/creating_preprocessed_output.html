Creating Preprocessed Output

If you identify a parse error that cannot be attributed to errors in your source code (for example, you checked your software build log and did not find the same errors, or if errors occur with viable code constructs that you know are not problematic for your environment), you can type -E on the command line to create a pre-processed output.

aset_CCcc -E --relative_path_base "/net/psales/space/apps/CosmosProject-1.2.4" --GNU_compatible_AST --c -I/net/psales/space/apps/CosmosProject-1.2.4 -D__svr4__ -D__SVR4 -D_GCC_NEW_VARARGS__ --gnu_asm --wchar_t /net/psales/space/apps/CosmosProject-1.2.4/bits.c > bits.out
