Submission Check Policy

The first task is to create a Submission Check policy for your site. The policy is stored in a policy file, the location of which is specified by the preference *psetPrefs.SubmissionCheck.Policy. If this preference is not set or if the file specified does not exist, Submission Check uses <CodeIntegrityInstalldir>/lib/baselib.sc as its default.

Note: The model build caches the preference values. For this reason, it is best to specify the path to the policy file (we recommend <repository>/prefs/baselib.sc) before running the model build.

If it is necessary to change the path of the policy file after the model build starts, the cached preference file, <repository>/tmp/disbuild/ mb.prefs, should be edited. This is necessary because the DIScover process started by the build server reads the cached preferences, not the original ones.

Note: Environment variables cannot be used in the cached preference file, so the <repository> value should be expanded in the preference in <repository>/prefs/baselib.sc.

The Submission Check policy has two parts:

•   a list of queries to run when checking files

•   a list of recipients for the generated reports
