Your Model Environment

Before you begin using DIScover, it is important for you to understand how your administrator set up the model environment, including which models are available on what servers.

The frequency that models are built with often reflects your development cycle and build schedule. For this reason, there may be many versions of information models available at one time. We recommend that you understand model naming and version schema so that you connect to the model that is most appropriate for your work.

A model server supports numerous sessions and clients. You can open the model you want to browse with the model browser, the lightweight command-line based Developer Xpress client, or the Access command-line interface.

The remainder of this book focuses on using the model browser; more information on Developer Xpress is presented in Developer Xpress and Access is presented in and the DIScover Access Online Help.
