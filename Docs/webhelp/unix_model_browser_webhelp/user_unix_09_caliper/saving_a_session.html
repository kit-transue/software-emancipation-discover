Saving a Session

After you run queries, you can save information from that session for future use. When you save sessions, Caliper records the following information:

•   your application’s total quality score

•   the quality score for each item

•   the values of any weights that you edited

•   the values of any threshold queries that you edited

Session information is stored as a group in a directory specified in the Group path.

The first time that you run queries, Caliper automatically saves the information in a session called default_run, unless you save the session with a different name.

Groups and status attributes for individual instances, however, are persistent from session to session. Any groups that you create while in that session appear in the Group Manager and remain until you delete them. Status attributes remain set until the information model is replaced with a new version.

Caliper does not save input information (the modules you ran queries on), so it is best to give your sessions descriptive names. Session names should not include special characters or spaces. Caliper flattens session names when saving; so avoid using case to distinguish sessions.
