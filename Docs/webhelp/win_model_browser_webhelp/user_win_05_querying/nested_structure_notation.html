Nested Structure Notation

The model browser displays the definition for a nested structure as it would for any entity type. As is standard in C++, DIScover displays the definition for a nested structure just as it would for inherited entities in C++.

For example, consider the following classes:

class base 0 {

class NSbase 0 {

int value;

public:

NSbase 0 (int x) {

value = x;

}

}

}

If you select the class base0 in the Results window, then select Show Nested or Show Nested Classes in the Queries pane, DIScover changes the display of the Results window to illustrate the class base0::NSbase0.

Once the query results appear, you can explore the data by invoking a graphical view of the entity (for more information, see ) or by double-clicking an entity to launch your IDE (for more information, see Working With IDEs).
